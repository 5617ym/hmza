# 1️⃣ الرؤية النهائية

بناء نظام احترافي يستطيع:

- استقبال أي ملف (PDF / صور / Excel / CSV / Word)
- استخراج المحتوى
- تحديد صفحات القوائم المالية فقط
- استخراج الجداول والأرقام منها
- إخراج JSON موحّد
- عرض النتائج في الواجهة

بدون انهيار عند:
- ملفات كبيرة
- ملفات مصورة
- تنسيقات مختلفة
- اختلاف اللغة

---

# 2️⃣ المعمارية المعتمدة (ثابتة)

Frontend  
↓  
/api/upload-url  
↓  
Azure Blob Storage  
↓  
/api/ingest (Router)  
↓  
مسار التحليل المناسب:  
- analyze (PDF / Images)  
- parse-xlsx  
- parse-csv  
- parse-docx  
↓  
Normalized JSON Output  
↓  
Frontend Display  

🚫 ممنوع تغيير هذه المعمارية أثناء التنفيذ.

---

# 3️⃣ القواعد الذهبية

1. لا ننتقل إلى مرحلة جديدة قبل نجاح الحالية.
2. لا نضيف ميزة جديدة قبل تثبيت الأساس.
3. أي خطأ = نصلحه في نفس الطبقة فقط.
4. كل مرحلة لها Definition of Done واضح.
5. لا تعديل معماري بدون توثيق داخل هذا الملف.

---

# 4️⃣ المراحل الرسمية

## 🟢 المرحلة 0 — تثبيت الهدف

تعريف شكل JSON النهائي:

{
  "fileName": "",
  "fileType": "",
  "pages": 0,
  "tables": 0,
  "textLength": 0,
  "sections": [],
  "numbers": [],
  "errors": []
}

✅ Done عندما نتفق أن هذا هو الشكل النهائي.

---

## 🟢 المرحلة 1 — رفع الملفات بثبات

### المطلوب
- /api/upload-url يعمل
- PUT إلى Blob ينجح
- الملف يظهر في Storage

### Definition of Done
- رفع ملف صغير
- رفع ملف كبير
- كلاهما يرجع 200/201 بدون خطأ

🚫 لا ننتقل قبل نجاح هذا.

---

## 🟢 المرحلة 2 — تحليل PDF/الصور (Document Intelligence)

### المطلوب
- /api/analyze يستقبل blobUrl
- يرجع:
  - pages
  - tables
  - textLength

### Definition of Done
- ملف PDF صغير يرجع أرقام صحيحة
- ملف PDF متوسط يرجع أرقام صحيحة
- لا يوجد 0 خاطئ
- لا يوجد JSON parse error
- لا يوجد 404

---

## 🟡 المرحلة 2B — معالجة الملفات الكبيرة (Fallback Engine)

إذا فشل التحليل بسبب الحجم:

- تقسيم PDF إلى أجزاء
أو
- تحليل بالصفحات
أو
- ضغط وتحويل

### Definition of Done
- ملف 60+ صفحة لا يفشل
- يرجع نتيجة مجمعة

---

## 🟢 المرحلة 3 — دعم Excel / CSV / Word

كل نوع له مسار خاص.

### Definition of Done
- ملف واحد من كل نوع يعمل

---

## 🟢 المرحلة 4 — Router (Ingest Layer)

`/api/ingest`

يحدد المسار حسب نوع الملف.

### Definition of Done
- لا يوجد 404
- كل نوع يذهب لمساره الصحيح

---

## 🟢 المرحلة 5 — اكتشاف القوائم المالية

### المطلوب
- البحث عن:
  - قائمة المركز المالي
  - قائمة الدخل
  - التدفقات النقدية
- تحديد صفحاتها فقط
- استخراج جداولها فقط

### Definition of Done
- تقرير مالي حقيقي
- يرجع pages محددة لكل قائمة

---

## 🟢 المرحلة 6 — توحيد الإخراج

إنتاج JSON موحّد بغض النظر عن المصدر.

---

## 🟢 المرحلة 7 — نظام اختبارات ثابت

5 ملفات مرجعية ثابتة:

1. PDF صغير نصي
2. PDF كبير
3. PDF ممسوح ضوئي
4. Excel
5. CSV

لا تعديل يمر قبل نجاحهم جميعًا.
